7) Data quality & notable issues

Observations and recommended clean-up steps:
  • Category label inconsistencies: some rows show "T﻿oys & Games" with an invisible/zero-width character and/or leading spaces (e.g., entries for ASINs B00003CCC3, B00012LLL2, B00018RRR8). Normalize by stripping whitespace and removing control/zero-width characters.
  • Currency: single value "USD" across rows — safe to keep as a constant or drop column if always USD and not needed.
  • Seller column mixes "Amazon" and "Third-Party:" patterns. Consider splitting into seller_type and seller_name for easier analysis (e.g., seller_type: Amazon/Third-Party, seller_name: vendor name).
  • ratings appear reasonable (4.0–4.9). reviews distribution is skewed (one very large outlier at 1560). Consider log-transforming reviews for modeling or visualization.
  • All ASINs unique; no duplicates detected.
  • No missing values found in this dataset snapshot.

8) Suggested next steps

  1. Normalize category strings (strip, remove control characters, consistent capitalization).
  2. Optionally split seller into seller_type and seller_name.
  3. Convert price & rating to numeric types and date_listed to datetime if not already done.
  4. Examine outliers in reviews and stock, and handle if needed for downstream modeling (capping or log transform).
  5. If building models, create features such as days_since_listed, is_fulfilled_by_amazon (seller == "Amazon"), and review_rate (reviews / (today - date_listed) if desired).
